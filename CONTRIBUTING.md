# Contributing to Songbird

Thanks for your interest in contributing!

## 🚀 What you can contribute

* Bug fixes
* Improvements to Docker setup
* Configuration optimizations
* Documentation updates
* Any original idea you have for the project

## 🛠 Getting Started

### Requirements

* Linux Ubuntu

### Run locally

```bash
npm run dev
```

Adjust configuration in `.env` if needed.

## 🌿 Branching Strategy

* Create branches from `main`
* Submit Pull Requests into `main`

## 📐 Guidelines

* Keep the app lightweight
* Do not hardcode secrets or credentials
* Use environment variables where possible
* Keep the code base clean and modular
* Keep configs clean and documented

## 📝 Commit Messages

Use clear commit prefixes:

* feat: new feature
* fix: bug fix
* docs: documentation changes
* refactor: code cleanup

## 🔁 Pull Requests

* Keep PRs small and focused
* Clearly describe your changes
* Test your changes before submitting
* Link related issues if applicable

## 🐛 Reporting Issues

Please include:

* Description of the problem
* Expected vs actual behavior
* Logs or error messages
* Environment details

## 🔐 Security

Do NOT submit secrets or private keys.
If you discover a vulnerability, please report it privately.
